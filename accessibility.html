This page's job is to teach accessibility while following every rule it describes: a correct heading order, a lang attribute on html, real landmarks, descriptive links, and a meaningful alt on its one image.

Descriptive Alt Text

Every meaningful image needs an alt attribute that describes what the image shows, not just its filename. Compare a good description to a lazy one:

A poor version of the same tag would read alt="icon1.svg", which tells a screen reader user nothing about what the image actually contains.

A Correct Heading Hierarchy

This page has exactly one h1, and every section below it is an h2. None of the headings skip a level, for example jumping from an h2 straight to an h4, because a skipped level breaks the outline that screen reader users rely on to navigate by heading.

The lang Attribute

The opening <html> tag on this very page is written as <html lang="en">, which tells screen readers and translation tools that the page's default language is English so text is pronounced correctly.

Semantic Landmarks

This page is wrapped in a header, a nav, this main region, and a footer below. Landmarks let a screen reader user open a list of regions and jump directly to the one they want instead of tabbing through the whole page in order.

Descriptive Link Text

Every link in this page's navigation says what it goes to, like "Read the Images page," instead of a vague phrase like "click here." Screen readers can list every link on a page out of context, so a link's own words need to make sense on their own.

Keyboard Navigation

The skip link at the very top of this page is the first thing a keyboard user reaches when pressing Tab, letting them jump straight to #main-content instead of tabbing through the entire navigation list on every single page. Try it: reload this page and press Tab once.
